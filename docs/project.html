map_info

Terminal UI 工具 — 浏览 Left 4 Dead 2 .vpk 插件文件，支持模糊搜索、地图代码过滤、重复检测

项目概况

map_info 是一个纯 Rust 构建的终端界面应用，帮助 L4D2 玩家管理和浏览 VPK 插件文件。 它解析 VPK 文件中的 missions/*.txt 内容，提取合作模式地图代码，并提供模糊搜索（带高亮）、 重复文件检测、地图代码列表（支持复制到剪贴板）等功能。启动时自动在后台扫描所有 VPK。 无任何外部本地依赖（不需要 .NET、不需要 C++ 运行时）。

功能特性

s 文件名模糊搜索 — Jaro-Winkler + Levenshtein 混合评分，匹配字符黄色高亮
c 地图代码过滤 — 按 modes → coop → Map 值搜索
d 重复文件检测 — Union-Find 算法找出共享地图的文件
f 地图代码列表 — 查看选中文件的合作地图列表，支持上下选择 + Enter 复制到剪贴板
t / r 排序切换 — 创建时间/修改时间、升序/降序
→ 查看 Mission 内容 — 带滚动百分比的可视化阅读
PgUp/PgDn 页面感知滚动 — 考虑换行文本的实际行数
后台扫描 启动时后台线程扫描全部 VPK，底部进度条实时显示，不影响界面操作
Toast 操作反馈 — 复制地图代码、清除过滤器等操作给出自动消失的浮动提示

架构设计

采用 Screen 驱动架构。事件处理和渲染根据当前 Screen 枚举值分发， 每个 Screen 有独立的事件处理器和渲染逻辑。新增界面只需添加枚举变体和对应模块。 

enum Screen {
    FileList,             // 默认：文件列表示
    Content,              // 查看 mission 内容
    Filter(FilterMode),   // 过滤对话框（文件名/地图代码）
    Duplicates,           // 重复文件弹窗
    MapCodes,             // 地图代码弹窗（可选中复制）
}
事件分发流程
// event_handler.rs — 非阻塞 poll(33ms)，全局键优先，然后按 Screen 分发
fn handle_events(app: &mut App) {
    // 1. 全局键（Ctrl+C, Ctrl+←/→, Ctrl+X）
    // 2. 按 Screen 分发
    match app.screen {
        Screen::Content    => content_handler(app, key),
        Screen::Filter(_)   => filter_handler(app, key),
        Screen::Duplicates => duplicates_handler(app, key),
        Screen::MapCodes   => map_codes_handler(app, key),
        Screen::FileList   => list_handler(app, key),
    }
}

数据流

.vpk 文件 → get_vpks() → IndexMap → App::init() → 后台线程扫描 → mpsc 事件 → map_cache → list_layout 渲染 → 用户选择文件 → VPKInfo::new() → content_layout 渲染

启动时 App::init() 调用 start_background_scan() 创建后台线程， 通过 mpsc::channel 发送 ScanEvent（Progress / MapEntry / Complete）。 主线程每帧通过 process_scan_events() 拉取事件更新 map_cache 和进度条。 此后筛选/去重/地图代码列表等操作直接使用缓存，无需重复解析。

技术细节

VPK 解析 — vpk.rs

  • 只支持 VPK v1 单文件格式（0x55AA1234 签名）
  • 解析目录树结构：扩展名 → 目录 → 文件名 → 18 字节条目
  • 读取文件数据时支持三种 archive_index：0x7FFF、0xFF7F、0（数据在同一个文件中）
  • 返回 &str，不克隆数据

KeyValues 解析 — mission.rs

  • 递归 tokenizer — 处理 // 注释、引号字符串、花括号
  • 提取 modes → coop 路径下的所有 Map 值
  • 忽略同层其他模式（versus、survival 等）

模糊匹配 & 高亮 — list_layout.rs

  • Jaro-Winkler 相似度 + Levenshtein 编辑距离混合评分
  • 子串命中按 匹配长度/文件名长度 加权加分，取代固定满分 1.0
  • 匹配位置系数：匹配越靠前分数越高，避免中间匹配与开头匹配并列
  • 平滑阈值：0.1 + (输入长度/12).min(0.7)，1 字符→0.18，4→0.43，8→0.77
  • 匹配字符在文件名中以黄色前景高亮显示（大小写不敏感）
  • 地图过滤时取所有 Map 值的最高分

重复检测 — app.rs

  • 基于 Union-Find 算法，按共享地图值合并文件组
  • 只显示至少包含一个重复地图值的文件组
  • 组内按文件名排序输出

后台扫描 — app.rs

  • 启动时在独立线程中扫描所有 VPK 文件
  • 通过 mpsc::channel 与主线程通信
  • 主线程每帧通过 try_recv() 非阻塞拉取事件
  • 底部状态栏右侧显示 Gauge 进度条
  • 扫描完成后进度条自动消失

Toast 通知 — app.rs

  • 统一通过 App::show_toast(msg) 调用
  • 在状态栏上方渲染暗底白字浮动条
  • 2 秒后自动消失
  • 用于复制地图代码、清除过滤器等操作反馈
